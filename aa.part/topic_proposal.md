# 项目提案：上市公司募集资金用途与变更跟踪的结构化抽取与动态分析研究

## 一、金融问题定义

上市公司在首次公开发行或再融资过程中需要披露募集资金的用途安排，但在后续经营过程中，这些资金用途可能因经营环境变化而发生调整、延期或变更。

这类信息通常分散在年度报告、临时公告以及募集资金使用专项报告中，且由于不同公司披露格式不统一，表述方式差异较大，导致募集资金用途难以进行标准化整理与横向比较。

同时，资金用途变更信息往往分散在不同时间点的公告中，缺乏连续性，使得投资者难以系统追踪资金实际使用路径及其偏离原始规划的程度。

基于此，本项目旨在对募集资金用途及其变更信息进行结构化抽取，实现：
1. 募集资金用途的标准化表达
2. 不同公司资金使用结构的可比分析
3. 资金用途变更的自动识别与记录
4. 资金使用偏离度的量化评估

## 五、跨文档匹配策略

### 匹配 Key 设计

采用三级匹配策略，确保跨文档一致性：

```
匹配 Key = company_code + "_" + fundraising_event_id + "_" + normalized_project_name
```

### 匹配规则
1. **公司代码**：使用6位股票代码作为基础标识
2. **募集资金批次**：通过公告中"募集资金说明"章节提取发行批次信息
3. **项目名称标准化**：
   - 去除标点符号和空格
   - 统一简称（如"有限公司"→"公司"）
   - 使用Levenshtein相似度进行模糊匹配（阈值>0.85）

## 七、难度档位声明

**档位选择：标准档（系数 1.0）**

### 难度依据
1. **单一复杂公告类型**：聚焦于"募集资金年度存放与使用情况专项报告"的解析
2. **字段数量**：13个核心字段，包含结构化数据和文本信息
3. **section检查**：针对专项报告中的"募投项目情况明细表"进行高确定性抽取
4. **证据回填**：每个字段均要求对应原文证据片段
5. **评估要求**：需要有人工样本验证和错误分类记录

### 不挑战更高档位的原因
- 暂不进行年报、专项报告、变更公告的多文档时间线串联
- 不涉及复杂的闭环分析和评分模型
- 专注于项目级字段的精准抽取与标准化

## 八、风险与备选方案

### 潜在风险
1. 不同公司专项报告中的"募投项目明细表"表格格式差异较大
2. PDF解析时表格内文字换行或多列错位可能导致数据错分
3. 项目名称简写或变动导致标准化对齐失误

### 备选方案
- 方案A：募集资金用途结构化抽取与标准化建模
- 方案B：行业资金使用偏离度对比分析
- 方案C：项目延期风险预测模型

## 九、最终输出形式

1. **结构化数据**：募集资金项目级 JSON/CSV 数据
2. **变更时间序列**：资金用途变更记录
3. **统计分析**：资金偏离度统计结果
4. **行业对比**：三家公司资金使用结构对比报告
5. **代码交付**：可复现代码与数据处理流程
6. **证据回溯**：抽取结果与原文的关联机制
